TI-Client: REST-API aktivieren

Aktivierung API TI-Client

Im Auslieferungszustand ist der TI-Client per Weboberfläche nur von localhost aus erreichbar. Ab Version 2.6.1 legt man in der Benutzerverwaltung mindestens einen Benutzer an. Der Benutzer hat als Administrator volle Zugriffsrechte via Weboberfläche wie wenn die Adminoberfläche per localhost aufgerufen wird.

Abbildung 1. Benutzerverwaltung auf dem TI-Client

Mit der Anlage des ersten Benutzers aktiviert man gleichzeitig das REST-API des TI-Clients. Danach legt man die Applikation cocard an. Das dabei kurz angezeigte Client-Secret notiert man und trägt es in Cocard beim TI-Client zusammen mit der Zugriffs-URL ein.
